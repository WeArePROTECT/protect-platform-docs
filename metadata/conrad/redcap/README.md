# REDCap Design and Specification Documentation

This directory contains design and specification documentation for the Conrad-owned REDCap database system.

## Status

REDCap is a Conrad-owned system currently **in construction**. Design is largely finalized; implementation is ongoing.

## Contents

This folder captures **design and specification documentation only** (not live data exports). All files here are non-sensitive design artifacts.

### Authoritative Documentation

**`redcap_file_package_readme.md`** — This is the authoritative REDCap file package README and should be treated as ground truth for REDCap design decisions, field specifications, and architecture.

### Included Documents

- **`redcap_file_package_readme.md`** — Authoritative REDCap file package documentation (ground truth)
- **`protect_redcap_metadata_readme.md`** — PROTECT REDCap metadata overview and architecture documentation
- **`REDCap Field Specification Version 7.10.2025.xlsx - Field Specification.csv`** — Historical REDCap field specification from July 2025 (never implemented; archived for reference)
- **`Protect Redcap Metadata Readme.docx`** — Word format of REDCap metadata overview
- **`Redcap File Package Readme.docx`** — Word format of REDCap file package documentation

## Purpose

These documents describe:
- REDCap database design and field specifications
- How REDCap relates to PROTECT server metadata
- Field inclusion/exclusion decisions
- Architecture and data flow expectations

## Note

This directory does **not** contain sensitive data exports. For information about monthly metadata drops (which are sensitive), see `../monthly_drops/`.
